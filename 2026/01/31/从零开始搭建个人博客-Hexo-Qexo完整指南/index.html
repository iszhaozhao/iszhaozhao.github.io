前言

本文记录了从零开始搭建个人博客的完整过程，包括 Hexo 静态博客框架的配置、主题应用、GitHub Pages 部署，以及 Qexo 在线编辑器的安装和配置。整个过程涵盖了从环境准备到最终上线的所有步骤。

技术栈

本次博客搭建使用了以下技术栈：

前端技术

  • Hexo 7.3.0 - 快速、简洁且强大的静态博客框架
  • hexo-theme-academia - 学术风格的 Hexo 主题
  • Node.js - JavaScript 运行时环境
  • Markdown - 文档编写格式

后端技术

  • Qexo 3.7.0 - 在线博客编辑器和管理系统
  • Django 3.2.25 - Python Web 框架
  • Python 3.12.2 - 编程语言

部署与版本控制

  • GitHub Pages - 静态网站托管服务
  • Git - 版本控制系统
  • GitHub - 代码托管平台

渲染引擎

  • hexo-renderer-pug - Pug 模板渲染器
  • hexo-renderer-stylus - Stylus CSS 预处理器
  • hexo-renderer-marked - Markdown 渲染器

七、常见问题解决

7.1 主题模板错误

问题: 浏览器显示 Pug 模板代码而不是渲染后的 HTML

解决:

  1. 安装 hexo-renderer-pug
  2. 修复 index.pug 中的 page.posts.data 访问方式
  3. 清理并重新生成

7.2 HTTPS 错误

问题: “You’re speaking plain HTTP to an SSL-enabled server port”

解决: 在 Django settings 中禁用 HTTPS 重定向：

7.3 Hexo 未检测到

问题: Qexo 提示 “Hexo not detected”

解决:

  • 确保配置的是源代码仓库，不是部署后的静态文件仓库
  • 检查仓库中是否包含 _config.yml、source、themes 等目录
